## AriS_運営 入口ガイド
このファイルから見れば、どこに何があるかすぐ分かる構成にしています。

## まず見る場所
- **基本情報を確認したい** → `01_基本情報/会社概要.md`
- **誰がどう対応するか確認したい** → `01_基本情報/対話体制_役割分担.md`
- **運営ルール・意思決定を確認したい** → `02_運用方針/運用方針.md`
- **投稿を作成したい** → `02_運用方針/投稿テンプレ_ショート動画.md`
- **明日の投稿案をそのまま使いたい** → `02_運用方針/投稿案_明日2本.md`
- **成果を記録したい** → `02_運用方針/KPI日次ログ.md`
- **週の実行手順を確認したい** → `02_運用方針/実行計画_今週.md`
- **今夜何をするか迷った** → `02_運用方針/実行パック_今夜.md`
- **自動化を回したい** → `automation/README_自動化.md`

## ディレクトリ構成
- `01_基本情報`
  - 会社の前提情報（形態 / 体制 / 組織図）
  - 対話体制（CEO / まほ）
- `02_運用方針`
  - 今週の意思決定
  - CEO執行方針（運用の判断基準）
  - 投稿テンプレ
  - 明日投稿案
  - KPI日次ログ
  - 週次実行計画
  - 当日実行パック
- `automation`
  - ショート生成スクリプト
  - YouTube投稿メタ生成
  - YouTube予約投稿スクリプト
  - 新しいPCでは `automation/setup_tools.ps1` を最初に1回実行

## 管理ルール（迷わないため）
- 新規ファイルは、原則 `01_基本情報` か `02_運用方針` のどちらかに置く
- どちらにも当てはまらない場合は、先にこのREADMEに置き場所を1行追記してから作成する
- ファイル名は「内容が分かる名詞」にする（例: `投稿方針.md`, `体制変更履歴.md`）
